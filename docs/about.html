A lightweight, real-time collaborative paste-sharing service built entirely in pure Java using only standard librariesâ€”no frameworks, no external dependencies. The application demonstrates advanced networking concepts including HTTP server implementation, WebSocket protocol for real-time updates, concurrent request handling, and persistent storage.

Features

  • Create & Share Pastes: Sequential ID generation (00001, 00002, etc.)
  • WebSocket Collaboration: Multiple users can edit simultaneously with real-time updates
  • File Persistence: JSON-based storage with atomic write operations
  • Public History: Browse all pastes with timestamps and previews
  • Soft Delete: Mark pastes as deleted while preserving data
  • Theme Toggle: Dark/light mode with localStorage persistence
  • Concurrency Control: Per-paste locking prevents data corruption

Network Concepts Used

  • Socket Programming: TCP server socket with client connection handling
  • TCP Lifecycle: Three-way handshake, connection management, graceful shutdown
  • HTTP Parsing: Custom HTTP/1.1 request/response parser and router
  • WebSocket Handshake: RFC 6455 upgrade protocol with Sec-WebSocket-Accept
  • Thread Pool: Fixed-size executor for concurrent client handling (10 threads)
  • File Locking: Per-ID ReentrantLock for safe concurrent paste updates
  • Atomic Writes: Temp file + rename pattern for crash-safe persistence

How to Run

# Start the server
chmod +x scripts/run.sh
./scripts/run.sh

# Server runs on http://localhost:8080

# Create a paste via curl
curl -X POST http://localhost:8080/create \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "text=Hello World"

# Read a paste (returns JSON)
curl http://localhost:8080/api/00001
